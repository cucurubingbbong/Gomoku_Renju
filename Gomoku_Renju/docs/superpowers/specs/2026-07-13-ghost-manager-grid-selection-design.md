# GhostManager 및 그리드 선택 개선 설계

## 목표

- 마우스가 가리키는 착수 가능한 교차점에 현재 차례의 반투명 돌을 표시한다.
- 바둑판 밖이나 이미 돌이 놓인 교차점에서는 고스트를 숨긴다.
- 바둑판 위치를 옮겨도 선택 위치, 고스트 위치, 실제 돌 위치가 일치하게 한다.
- 기존 매니저 중심 구조와 한국어 주석 스타일을 유지한다.

## 좌표 변환

`GridManager`가 그리드 좌표 변환의 단일 기준이 된다. 바둑판 중앙 `Transform`을 참조하고 `(gridSize - 1) / 2`를 이용해 월드 좌표와 그리드 좌표를 양방향으로 변환한다. 이번 변경은 바둑판 위치 이동을 지원하며 회전과 셀 간격 변경은 범위에 포함하지 않는다.

`PlaceManager`는 레이캐스트 충돌 지점을 받은 즉시 `GridManager`를 통해 그리드 좌표로 변환한다. 범위를 벗어난 좌표는 선택 실패로 처리한다. 고스트와 실제 돌은 동일한 그리드-월드 변환을 사용해 같은 교차점에 배치한다.

## GhostManager

`GhostManager`는 고스트 돌의 생성, 표시, 이동, 숨김을 담당한다. 현재 차례가 바뀌면 필요한 돌 프리팹으로 고스트를 교체한다. 고스트의 콜라이더를 비활성화하고 복제한 런타임 머티리얼을 반투명으로 설정하여 실제 레이캐스트와 기존 머티리얼에 영향을 주지 않는다.

`PlaceManager`는 선택한 칸이 착수 가능하면 고스트를 표시하고, 착수 불가능하거나 레이캐스트가 실패하면 숨긴다. 착수 직전에도 고스트를 숨겨 실제 돌과 겹쳐 보이지 않게 한다.

## 돌 프리팹 순서

기존 `stones` 배열 순서인 `0: 백돌`, `1: 흑돌`을 유지한다. `StoneType` 값을 단순히 1만큼 빼지 않고 명시적인 변환 메서드를 사용해 현재 반대로 선택되는 색상 문제를 수정한다.

## 오류 처리

- 필수 참조가 없으면 불명확한 위치에 돌을 만들지 않는다.
- 선택 좌표가 그리드 범위를 벗어나면 고스트를 숨기고 착수를 막는다.
- 고스트 오브젝트의 모든 콜라이더를 비활성화해 선택 레이캐스트를 방해하지 않게 한다.

## 검증

현재 프로젝트에는 자동화 테스트 어셈블리가 없으므로 새 테스트 인프라는 추가하지 않는다. C# 솔루션 빌드로 컴파일을 확인하고 다음 플레이 모드 시나리오를 점검한다.

1. 빈 교차점에서 현재 차례 색상의 고스트가 표시된다.
2. 마우스를 판 밖이나 점유된 칸으로 옮기면 고스트가 숨겨진다.
3. 클릭한 교차점에 고스트와 동일한 색상 및 위치의 실제 돌이 놓인다.
4. `GomokuBoard` 위치를 옮겨도 선택, 고스트, 실제 돌이 함께 이동한다.
